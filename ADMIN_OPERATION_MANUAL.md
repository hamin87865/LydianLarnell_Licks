# Lydian Larnell Licks 관리자 운영 매뉴얼

## 1. 관리자 로그인
1. 관리자 계정으로 로그인합니다.
2. 상단 Navbar에서 `관리자`, `정산메뉴`, `마이페이지`, `홈`이 보이면 관리자 권한이 정상입니다.
3. 로그인 실패 시 먼저 이메일/비밀번호를 다시 확인하고, 계속 실패하면 DB의 사용자 role 값이 `admin`인지 확인합니다.

## 2. 뮤지션 승인 절차
1. `관리자` 메뉴로 이동합니다.
2. 대기 중 신청 목록에서 신청자 정보를 확인합니다.
3. 제출 영상, 계약서, 계좌정보(마스킹)를 검토합니다.
4. 승인 시 `승인`, 거절 시 거절 사유와 관리자 메모를 입력 후 `거절`을 누릅니다.
5. 처리 후 목록이 갱신되고 감사 로그에 기록되어야 정상입니다.

## 3. 콘텐츠 관리 방법
1. 카테고리 페이지 또는 관리자 화면에서 콘텐츠를 확인합니다.
2. 관리자 전용 메뉴로 삭제 또는 제재를 수행합니다.
3. 제재 시 사유를 반드시 남깁니다.
4. 제재/해제/삭제 후 사용자 화면에 즉시 반영되는지 확인합니다.

## 4. 결제 처리 점검 방법
1. 결제는 `/api/payments/prepare` → Toss 결제창 → `/api/payments/confirm` 순서로만 완료됩니다.
2. 동일 콘텐츠를 이미 구매한 사용자는 다시 구매되지 않아야 합니다.
3. 중복 승인 의심 시 관리자 감사 로그와 결제 로그를 먼저 확인합니다.
4. 주문 상태 확인이 필요하면 `/api/payments/status?orderId=...` 로 상태를 조회합니다.

## 5. 정산 처리 방법
1. 상단 `정산메뉴`로 이동합니다.
2. 월을 선택해 해당 월 정산내역을 확인합니다.
3. 기본 상태는 `지급대기`, 지급 처리 후 `지급완료`로 바뀌어야 합니다.
4. `지급완료 처리`는 한 번만 처리되어야 하며, 중복 클릭 시 추가 지급이 되면 안 됩니다.
5. 계좌번호 전체 조회는 관리자만 가능합니다.
6. 지급 후 감사 로그와 월 정산 스냅샷이 저장되어야 정상입니다.

## 6. 장애 발생 시 대응 절차
1. `/healthz` 와 `/readyz`를 확인합니다.
2. Render 서비스 로그에서 최근 오류를 확인합니다.
3. Sentry와 관리자 알림 이메일 수신 여부를 확인합니다.
4. 결제 장애는 `payment_audit_logs`, 운영 장애는 `admin_audit_logs`를 먼저 확인합니다.
5. DB 장애 또는 데이터 손상 의심 시 Render PostgreSQL 백업 시점과 복구 절차를 확인합니다.

## 7. 서버 재시작 후 점검
1. 로그인 정상 여부
2. 관리자 메뉴 진입 여부
3. 업로드 파일 유지 여부
4. 결제 준비 API 응답 여부
5. 정산내역 조회 여부
6. `/readyz` DB 연결 여부
